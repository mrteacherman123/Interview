① Breadth-First Search
1 BFS — Level-by-Level Exploration
BFS expansion — each ring is one level deeper
Java template
// Standard BFS template
Queue<T> q = new LinkedList<>();
Set<T> visited = new HashSet<>();
q.add(start);
visited.add(start);

while (!q.isEmpty()) {
  T node = q.poll();
  // process node
  for (T neighbor : getNeighbors(node)) {
    if (!visited.contains(neighbor)) {
      visited.add(neighbor);
      q.add(neighbor);
    }
  }
}
// Level-order variant:
// freeze q.size() per layer
complexity
O(V + E) time · O(V) space
Each vertex is processed once, and each edge is examined once. The queue and visited structure each grow to at most V.
when to use
Shortest path, level order, minimum steps
Word Ladder, Walls and Gates, 01 Matrix, Rotting Oranges, shortest path in an unweighted graph or grid.
key gotcha
Visit on enqueue, not on poll
If you only mark visited when removing nodes from the queue, the same node can be enqueued many times and blow up runtime.
② Depth-First Search
2 DFS — Deep Dive then Backtrack
DFS call stack — explore fully before backtracking
Java template
// Recursive DFS on a grid
void dfs(int r, int c, boolean[][] vis) {
  vis[r][c] = true;
  int[] dr = {1, -1, 0, 0};
  int[] dc = {0, 0, 1, -1};
  for (int i = 0; i < 4; i++) {
    int nr = r + dr[i], nc = c + dc[i];
    if (valid(nr, nc) && !vis[nr][nc]) {
      dfs(nr, nc, vis);
    }
  }
}
when to use
Connected components, flood fill, cycle detection
Number of Islands, Clone Graph, Pacific Atlantic, Surrounded Regions, and directed-cycle checks with a recursion state.
DFS flood-fill on grid (Number of Islands pattern)
③ Topological Sort
3 Topological Sort — Order a DAG
Kahn's algorithm (BFS-based) — indegree countdown
step 1
Build indegree map
For every directed edge u → v, increment indegree[v]. Include nodes with zero indegree too.
step 2
Seed the zero-indegree queue
These nodes have no prerequisites, so they can appear earliest in the ordering.
step 3
Peel layers until empty
Processing a node removes its outgoing edges. Any neighbor that falls to zero indegree becomes newly available.
④ Union Find (Disjoint Set Union)
4 Union Find — Merge & Query Components
Union with path compression — parent array and tree shape
Union Find — Java
class UF {
  int[] parent, size;

  UF(int n) {
    parent = new int[n];
    size = new int[n];
    for (int i = 0; i < n; i++) {
      parent[i] = i;
      size[i] = 1;
    }
  }

  int find(int p) {
    if (parent[p] != p) {
      parent[p] = find(parent[p]);
    }
    return parent[p];
  }

  boolean union(int p, int q) {
    int rp = find(p), rq = find(q);
    if (rp == rq) return false;
    if (size[rp] < size[rq]) {
      int tmp = rp; rp = rq; rq = tmp;
    }
    parent[rq] = rp;
    size[rp] += size[rq];
    return true;
  }
}
complexity
≈ O(α(N)) per op
With path compression and union by size or rank, operations are effectively constant for interview-scale input sizes.
when to use
Dynamic connectivity, group merging
Redundant Connection, Number of Provinces, Similar String Groups, Kruskal MST, or online addEdge() queries.
vs DFS/BFS
Online, not traversal-based
DFS and BFS traverse an existing graph. Union Find excels when edges arrive incrementally and you only care about component membership.
⑤ Multi-Source BFS
5 Multi-Source BFS — Simultaneously from All Sources
Wavefront view — all sources start at distance 0 together
Multi-source BFS init
// 1. Seed ALL sources at once
for (int i = 0; i < m; i++) {
  for (int j = 0; j < n; j++) {
    if (mat[i][j] == 0) q.add(new int[] {i, j});
    else mat[i][j] = Integer.MAX_VALUE;
  }
}

// 2. Standard BFS from all seeds
while (!q.isEmpty()) {
  int[] cur = q.poll();
  for (int d = 0; d < 4; d++) {
    int nr = cur[0] + dr[d], nc = cur[1] + dc[d];
    if (valid(nr, nc) &&
        mat[nr][nc] > mat[cur[0]][cur[1]] + 1) {
      mat[nr][nc] = mat[cur[0]][cur[1]] + 1;
      q.add(new int[] {nr, nc});
    }
  }
}
key insight
Do not run BFS separately from each source
That repeats work and costs O(k × V). Simultaneous seeding lets the first wave that reaches a cell define its optimal nearest-source distance.
⑥ Dijkstra's Algorithm
6 Dijkstra — Shortest Path with Weighted Edges
Priority queue extracts the current minimum-distance node
Dijkstra — Java
// Shortest path in weighted graph
int[] dist = new int[n];
Arrays.fill(dist, Integer.MAX_VALUE);
dist[src] = 0;

PriorityQueue<int[]> pq =
  new PriorityQueue<>((a, b) -> a[0] - b[0]);
pq.add(new int[] {0, src});

while (!pq.isEmpty()) {
  int[] cur = pq.poll();
  int d = cur[0], u = cur[1];
  if (d > dist[u]) continue; // stale heap entry

  for (int[] edge : adj[u]) {
    int v = edge[0], w = edge[1];
    if (dist[u] + w < dist[v]) {
      dist[v] = dist[u] + w;
      pq.add(new int[] {dist[v], v});
    }
  }
}
Key distinction: BFS assumes all edges cost the same, so “first seen” means optimal. Dijkstra replaces the FIFO queue with a min-heap so the next node processed is always the globally cheapest tentative path.
⑦ Bipartite Check
7 Bipartite Check — Graph 2-Coloring
Alternate colors across edges — any same-color edge breaks bipartiteness
definition
Split nodes into 2 sets, edges only cross sets
A graph is bipartite iff it has no odd cycle. That is exactly the same as saying it can be 2-colored.
when to use
Conflict graphs, rivalries, 2-group assignment
Use it for “can these be split into two compatible groups?” questions. Standard solution is BFS or DFS coloring.
union find variant
Group each node's neighbors together
For every node u, all of its neighbors must live in the opposite partition. If find(u) == find(v) for some edge, the graph is not bipartite.
