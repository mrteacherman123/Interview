② Traversal Template
2 The Core DFS Frame
mental-template.java 
State dfs(TreeNode node) {
  if (node == null) return base;

  State left = dfs(node.left);
  State right = dfs(node.right);

  // 1) compute answer at this node
  // 2) maybe update global result
  // 3) return state parent actually needs

  return merged(left, right, node.val);
}
Rule
Separate Local Answer vs Returned State
In tree DP, the value you return upward is often not the full answer for the subtree.

For example, maximum path sum updates the global answer with left + node + right, but returns only one downward branch.

④ Pattern Library
4 Five Patterns That Cover Most Tree Interviews
Pattern
Level Order + Layer Logic
Use a queue when the prompt says per level, left to right, zigzag, largest per row, or right side view.
Pattern
Postorder Return Tuple
Return multiple values when the parent needs more than one fact: examples include min/max/size for largest BST subtree.
Pattern
Global Best + Local Return
Classic when a full path or full subtree answer differs from the one-branch state the parent can legally use.
Pattern
Rebuild With Boundaries
Tree construction problems are almost always “one traversal gives root order, the other gives split position”.
Pattern
BST Candidate Walk
For predecessor, successor, LCA, and trim problems, ancestor candidates are updated only on one direction of turn.
Pattern
Shape Preservation
Serialization, same-tree, and build problems are about preserving exact structure, not only node values.
⑤ Checklist
5 Before You Code
Interview Checklist
Ask These Questions First
Most tree bugs are not algorithmic. They are state-definition bugs.
What exactly does my recursive call return to the parent?
Do I need a global answer separate from the returned value?
Is this really a BST shortcut problem or a general-tree problem?
Does equality break the invariant? BST validation usually needs strict inequality.
Am I counting nodes, edges, levels, or paths?
If I rebuild or serialize, how is the exact shape preserved?
