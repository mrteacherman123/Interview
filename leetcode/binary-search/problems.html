Answer-Space Search Monotonic Predicate ok(mid) First True / Last True Boundary Playbook ← Pattern Guide
Core Idea
Search answers, not just arrays.

A lot of "binary search" problems are not really about finding a target index. They are about searching a sorted space of possible answers and asking whether a guess mid is valid. If ok(mid) flips only once, binary search can find that boundary quickly.

speed capacity divisor partition pivot window start
01 / Spot It
Search space is conceptual
You are guessing speed, capacity, divisor, partition size, or boundary index rather than looking for an exact element.
02 / Ask
Can mid work?
Write ok(mid) so it returns yes or no. The predicate must flip only once across the whole search domain.
03 / Shrink
Keep the right boundary
If you want minimum feasible, keep the first true. If you want maximum valid, keep the last true.
Boundary Template

First true = minimum feasible

Use when the predicate looks like FFFFTTTT. This is the common "smallest speed / capacity / divisor that works" form.

while (lo < hi) {
  mid = lo + (hi - lo) / 2;
  if (ok(mid)) hi = mid;
  else lo = mid + 1;
}

Last true = maximum valid

Use when the predicate looks like TTTTFFFF. This is the "largest answer that still works" form.

while (lo + 1 < hi) {
  mid = lo + (hi - lo) / 2;
  if (ok(mid)) lo = mid;
  else hi = mid;
}

How to pick lo and hi

Pick bounds that make your invariant obvious. The real question is not "what are the smallest and biggest numbers?" It is "what does lo mean, and what does hi mean, before the loop starts?"

Sometimes both ends are valid candidates. Sometimes one side is just a sentinel boundary used to keep the false/true split clean.

Both ends are real candidates Use this when the search domain itself is already clean. For sqrt(x), lo = 0 is a definitely valid answer and hi = x/2 + 1 is a definitely invalid upper cap. For k-closest elements, the only legal window starts are 0..n-k, so use lo = 0, hi = n-k.
sqrt(x) valid ........ answer ........ invalid lo=0 hi=x/2+1 k-closest windows [0] [1] [2] ... [n-k] ^ ^ lo hi
One side is a virtual sentinel Use this when you want a guaranteed false/true boundary just outside the array. For rotated minimum, lo = -1 means "virtual false before index 0" and hi = n-1 is guaranteed true, so returning hi makes sense. Likewise, lower-bound searches often use hi = n as a virtual right boundary.
rotated minimum index: -1 0 1 2 3 4 ok(): F F F T T T ^ ^ lo=-1 hi=first T lo is fence, not answer
Return the side that stores the answer If the invariant is "lo = last valid", return lo. If the invariant is "hi = first valid" or "hi = surviving candidate", return hi. The other side may exist only to fence the search.
last valid T T T F F -> return lo first valid F F F T T -> return hi Ask: which pointer is carrying the boundary?
Rule of thumb: write the invariant in one sentence before coding. Example: "For sqrt, lo is valid and hi is invalid." Example: "For rotated minimum, lo is on the false side and hi is on the true side." Once that sentence is clear, the initial values usually become obvious.
Pattern Atlas
Map the problem before you code.

Search a value range

Guess an answer and check feasibility or count how many values land on one side of mid.

Search a boundary

The data is already sorted, or can be treated like it is. You are really finding a left edge, right edge, or valid window start.

#7 first/last, #9 k closest, #10 2D matrix, #15 unknown size

Slope, pivot, or partition

The signal comes from local structure: sorted half, descending slope, pivot side, or a partition validity check.

#3 peak, #4 rotated search, #5 rotated min, #8 median, #14 mountain peak

Halving as the main weapon

These are binary-search-adjacent. The win still comes from repeatedly cutting the search or recursion space in half.

#6 pow, #12 kth smallest unsorted, #13 coincidence search
Pitfalls Checklist
Decide the boundary before the loop. Ask whether you want the first true, last true, or an exact hit. That choice determines the loop shape and whether you return lo or hi.
Make the predicate monotonic. If ok(mid) can flip back and forth, binary search is the wrong tool. Feasibility must move in one direction only.
Search the right domain. Some problems search indices, others search answer values like speed, capacity, divisor, or partition count.
Respect sentinels and overflow. Use long for mid * mid, watch hi = n sentinel cases, and be explicit about virtual boundaries like -∞.
01 / Matrix
Kth Smallest Element in Sorted Matrix
Binary Search Priority Queue

n×n matrix where rows & columns are both sorted. Find the kth smallest value overall — not the kth distinct element.

Key Insight
Search over value range [matrix[0][0] … matrix[n-1][n-1]], not indices. For each mid, count elements ≤ mid using a two-pointer sweep row-by-row.
Invariant
Any value below lo is still too small, and hi is kept as a value whose count of elements ≤ it is already at least k.
// count elements ≤ mid
int j=n-1, count=0;
for(int i=0; i<m; i++) {
  while(j>=0 && matrix[i][j]>mid) j--;
  count += (j+1);
}
// if count < k → lo=mid+1
// else        → hi=mid
⚠ Gotcha
j must not reset per row — carry it across rows for O(m+n) counting. The answer always exists in the matrix (no need for contains check).
Time: O((m+n) log(max−min)) Space: O(1)
range search count ≤ k 2D matrix
02 / Math
Integer Square Root — Sqrt(x)
Binary Search

Given a non-negative integer x, return its integer square root (truncate decimals). No built-in sqrt allowed.

Key Insight
Search over [2 … x/2]. Predicate: mid*mid ≤ x. Find the largest mid where this is true → use lo = mid pattern (TTTT'T'FFFF).
Invariant
lo is a valid square root candidate with lo*lo ≤ x, and hi is invalid with hi*hi > x.
long lo=0, hi=x/2+1;
while(lo < hi-1) {
  long mid = lo + (hi-lo)/2;
  if(mid*mid <= x) lo = mid;
  else            hi = mid;
}
return (int)lo; // largest valid lo
⚠ Gotcha
Cast mid*mid to long to avoid overflow. Handle x ≤ 1 separately. Return lo (not hi) — seeking the largest truthy value.
Time: O(log x) Space: O(1)
largest valid TTFF boundary overflow trap
03 / Array
Find Peak Element
Binary Search Recursive

Array where nums[i] ≠ nums[i+1]. Find any index where the element is greater than both its neighbors. Multiple peaks may exist.

Key Insight
If nums[mid] > nums[mid+1], a peak exists on the left (including mid). Otherwise, it exists strictly to the right. Boundaries: nums[-1]=nums[n]=−∞ guarantees a peak always exists.
Invariant
The peak index always stays in (l, r]; l is kept on the rising side, and r stays on a side that still contains a peak.
int l=-1, r=nums.length-1;
while(l+1 < r) {
  int mid = (l+r)/2;
  if(nums[mid] > nums[mid+1]) r=mid;
  else                         l=mid;
}
return r; // hi holds peak
⚠ Gotcha
Return r (hi), not l. The predicate ok(mid) = nums[mid] > nums[mid+1] is TTTT'T'FFFF: we seek last T → return r.
Time: O(log n) Space: O(1)
local condition return hi virtual boundaries
04 / Rotated
Search in Rotated Sorted Array
Two-Phase BS Single Pass

Sorted array rotated at an unknown pivot (e.g. [4,5,6,7,0,1,2]). Find the index of a target, or -1. Must be O(log n).

Key Insight (Two-Phase)
Phase 1: Find pivot (min index) using nums[mid] ≤ nums[last].
Phase 2: Target ≥ nums[0]? Binary search left half. Else right half.
Invariant
Phase 1 keeps the pivot inside the surviving range; phase 2 keeps the target, if it exists, inside exactly one sorted half.
// Single-pass approach
if(nums[mid]>=nums[start]) {
  // left half is sorted
  if(target>=nums[start] &&
     target<nums[mid])  end=mid-1;
  else                 start=mid+1;
} else {
  // right half is sorted
  if(target>nums[mid] &&
     target<=nums[end]) start=mid+1;
  else                 end=mid-1;
}
⚠ Gotcha
The predicate for finding pivot is nums[mid] ≤ nums[last] → FFFF'T'TTTT. After finding pivot, do standard binary search on the correct half.
Time: O(log n) Space: O(1)
find pivot first half sorted two phases
05 / Rotated
Find Minimum in Rotated Sorted Array
Binary Search

Same rotated array as #4. This time just find the minimum element (which is the pivot point) in O(log n).

Key Insight
Predicate ok(mid): is nums[mid] ≤ nums[last]? This is FFFF'T'TTTT. We want the first true → use hi = mid and return hi at end.
Invariant
lo is on the false side or is the sentinel -1, while hi is on the true side and still contains the minimum index.
int lo=-1, hi=nums.length-1;
while(hi > lo+1) {
  int mid = lo + (hi-lo)/2;
  // FF'T'TTTT → seek first T
  if(nums[mid]<=nums[nums.length-1])
       hi = mid;
  else lo = mid;
}
return nums[hi]; // hi = pivot = min
⚠ Gotcha
Compare against nums[last] (not nums[lo]). For non-rotated arrays (nums[0]<nums[n-1]), the answer is simply nums[0] — handle as special case optionally.
Time: O(log n) Space: O(1)
first true FFTT boundary compare to last
06 / Math
Pow(x, n) — Fast Exponentiation
Divide & Conquer Bit Manipulation

Implement pow(x, n) for any real x and integer n (including negatives). Must run in O(log n), not O(n).

Key Insight
Halve the problem each step: x^n = half*half if n even, half*half*x if odd. This achieves O(log n). For negative n: return 1 / power(x, |n|).
Invariant
Each recursive call still represents the same mathematical goal, x^n; halving n preserves the answer and only changes how you recombine it.
double power(double x, int n) {
  if(n==0) return 1;
  double half = power(x, n/2);
  if(n%2==0) return half*half;
  else        return half*half*x;
}
// caller: n<0 → 1/power(x,|n|)
⚠ Gotcha
n = Integer.MIN_VALUE: negating it overflows. Fix: n = n+1 before negating, then multiply extra x at end. Also watch iterative: accumulate into res on odd steps.
Time: O(log n) Space: O(log n) stack
halving negative n INT_MIN edge
07 / Array
First & Last Position of Element
2× Binary Search

Sorted array with duplicates. Return the [start, end] indices of a target value. Return [-1,-1] if not found. O(log n).

Key Insight
Run binary search twice with different predicates. Left bound: nums[mid] ≥ target → first occurrence. Right bound: nums[mid] > target → one past last occurrence, subtract 1.
Invariant
Left search keeps the first index ≥ target in [lo, hi), and right search keeps the first index > target in [lo, hi).
// unified helper: left=true → first
if(nums[mid]>target ||
   (left && nums[mid]==target))
     hi = mid;
else
     lo = mid+1;
// right bound: result-1
// hi=nums.length (not length-1!)
⚠ Gotcha
Set hi = nums.length (not length-1) to handle targets at the very end. After finding left bound, validate nums[left]==target before computing right.
Time: O(log n) Space: O(1)
lower bound upper bound hi=n sentinel
08 / Hard
Median of Two Sorted Arrays
Partition BS

Two sorted arrays of sizes m and n. Find the median of the combined array in O(log(m+n)) — no merging allowed.

Key Insight
Binary search on partition point of smaller array. Set partY = (x+y+1)/2 − partX. Valid partition: maxLeftX≤minRightY AND maxLeftY≤minRightX.
Invariant
The correct partition is always searched in the smaller array; a valid split keeps exactly half the elements on the left and every left value ≤ every right value.
int partX = (lo+hi)/2;
int partY = (x+y+1)/2 - partX;
// edge guards:
maxLX = partX==0? INT_MIN : A[partX-1];
minRX = partX==x  ? INT_MAX : A[partX];
// valid → compute median
// maxLX>minRY → hi=partX-1
// else        → lo=partX+1
⚠ Gotcha
Always binary search on the smaller array. The +1 in (x+y+1)/2 correctly handles both even and odd combined lengths. Use INT_MIN/MAX for empty partition guards.
Time: O(log min(m,n)) Space: O(1)
partition search even+odd trick smaller array
09 / Array
Find K Closest Elements
Sliding Window BS PQ O(nlogk)

Sorted array, integers k and x. Return the k numbers closest to x, sorted. Ties go to the smaller element.

Key Insight (Best O(log n))
Find left boundary of a window of size k. Predicate: x−arr[mid] > arr[mid+k]−x (raw, not abs!) means window should slide right. Search space: [0, n−k].
Invariant
[lo, hi] contains only legal window starts, and once a start is proven too far left, every start ≤ it can be discarded too.
int lo=0, hi=arr.length-k;
while(lo < hi) {
  int mid = (lo+hi)/2;
  // do NOT use abs() here!
  if(x-arr[mid] > arr[mid+k]-x)
       lo = mid+1;
  else hi = mid;
}
// return arr[lo .. lo+k)
⚠ Gotcha
Never use abs() in the comparison — it fails when arr has duplicates like [1,1,2,2,2,2,2,3,3], x=3. Raw signed comparison preserves directional information.
Time: O(log(n−k) + k) Space: O(k)
window left boundary no abs trap hi=n-k
10 / Matrix
Search a 2D Matrix
Virtual 1D BS

m×n matrix where each row is sorted and the first element of each row is greater than the last of the previous. Search for a target.

Key Insight
Treat the m×n matrix as a virtual sorted 1D array of length m*n. Map index: row = idx/n, col = idx%n. Binary search over [0 … m*n−1].
Invariant
The target, if present, stays inside the surviving virtual 1D range, and each mapped comparison removes half of those virtual positions.
int lo=0, hi=m*n;
while(lo+1<hi) {
  int mid = lo + (hi-lo)/2;
  // map back: row=mid/n col=mid%n
  if(matrix[mid/n][mid%n]<=target)
       lo=mid;
  else hi=mid;
}
return matrix[lo/n][lo%n]==target;
⚠ Gotcha
Guard against empty matrix: check m==0 || n==0 first. The predicate is TTTTT'T'FFFF — we seek last true → check lo at end.
Time: O(log(m·n)) Space: O(1)
index mapping virtual array row/col formula
11 / Math
Kth Missing Positive Number
Count Formula BS

Strictly increasing array of positive integers with gaps. Find the kth positive integer that is missing from it.

Key Insight
At index pivot, the number of missing positives before arr[pivot] is arr[pivot] − pivot − 1. Binary search for the rightmost pivot where this count < k. Answer: left + k.
Invariant
Indices left of left all have missing-count k, and indices right of right all have missing-count ≥ k.
int left=0, right=arr.length-1;
while(left<=right) {
  int p = left+(right-left)/2;
  // missing before arr[p]:
  if(arr[p]-p-1 < k) left=p+1;
  else                right=p-1;
}
return left + k; // elegant formula
⚠ Gotcha
The return formula left + k is derived: missing before arr[right] is arr[right]−right−1, so answer = arr[right] + k − (arr[right]−right−1) = k + left. Don't try to re-derive from scratch.
Time: O(log n) Space: O(1)
count formula left+k trick missing count
12 / Select
Kth Smallest Element (Unsorted)
Quickselect

Unsorted array of integers. Return the kth smallest (0-indexed) in O(n) average time — no full sort allowed.

Key Insight
Quickselect partitions around a pivot and recurses only into the half containing index k — O(n) average. Randomize first to avoid O(n²) worst case on sorted input.
Invariant
After partition, pivot index j is in its final sorted position, and the target index k stays inside the surviving range [lo, hi].
// partition: nums[hi] is pivot
while(lo<hi) {
  int j = partition(nums, lo, hi);
  if(j<k)       lo=j+1;
  else if(j>k)  hi=j-1;
  else           break;
}
return nums[k];
⚠ Gotcha
Always randomize (Fisher-Yates shuffle) before partitioning. Without randomization, sorted inputs cause O(n²). Partition uses nums[hi] as pivot and swaps elements ≤ pivot to the left.
Time: O(n) avg O(n²) worst
quickselect randomize first partition
13 / Simulation
Coincidence Search
Recursive Sim

Unsorted array of unique integers. Count how many elements would still be found correctly if binary search were applied to it as-is.

Key Insight
Simulate binary search structurally. Track [ll, hh] — the valid range for a correct element at each position. If ll ≤ nums[mid] ≤ hh, binary search would find it. Recurse, tightening bounds.
Invariant
Each recursive frame carries both the index range binary search would visit and the value range [ll, hh] that still makes an element findable there.
def bsearch(lo, hi, ll, hh):
  if lo <= hi:
    mid = lo + hi >> 1
    if ll <= nums[mid] <= hh:
      ans += 1
    bsearch(lo, mid-1, ll, min(hh,nums[mid]))
    bsearch(mid+1, hi, max(ll,nums[mid]), hh)
⚠ Gotcha
When going left, the new upper bound tightens: hh = min(hh, nums[mid]). When going right: ll = max(ll, nums[mid]). Recursion depth is O(log n) — not O(n).
Time: O(n) Space: O(log n)
simulate BS valid range tighten bounds
14 / Mountain
Peak Index in a Mountain Array
Binary Search

Array guaranteed to be a "mountain": strictly increases then strictly decreases. Find the index of the peak element.

Key Insight
Identical to #3 (Find Peak Element) but guaranteed mountain shape. Predicate arr[mid] ≥ arr[mid+1] is TTTT'T'FFFF. Seek first true → hi = mid, return hi.
Invariant
lo stays on the ascending side, hi stays on the descending side or at the peak, and the peak index remains in (lo, hi].
int lo=-1, hi=arr.length-1;
while(lo+1<hi) {
  int mid = lo + (hi-lo)/2;
  if(arr[mid]>=arr[mid+1])
       hi=mid; // on descent side
  else lo=mid; // still ascending
}
return hi; // peak index
⚠ Gotcha
Mountain is guaranteed, so no boundary checks needed for mid+1. The guarantee means the peak is always strictly interior — lo=-1, hi=length-1 safely excludes endpoints.
Time: O(log n) Space: O(1)
slope direction guaranteed peak return hi
15 / Unbounded
Search in Sorted Array of Unknown Size
Unbounded BS

Sorted array accessible only via an ArrayReader API (no .length). Out-of-bounds reads return INT_MAX. Find the target index.

Key Insight
Use hi = Integer.MAX_VALUE as sentinel. Predicate ok(mid): reader.get(mid) is not 2147483647 (OOB sentinel) AND ≤ target. Pattern: TTTTT'T'FFFF.
Invariant
lo is a reachable index with value ≤ target, while hi is either out of bounds or points to a value > target.
int lo=0, hi=Integer.MAX_VALUE;
while(lo+1<hi) {
  int mid = lo + (hi-lo)/2;
  if(ok(reader, mid, target))
       lo=mid;
  else hi=mid;
}
// ok: !OOB && val<=target
return reader.get(lo)==target? lo:-1;
⚠ Gotcha
Must check OOB sentinel (2147483647) in predicate before value comparison — short-circuit order matters. With hi=MAX_VALUE, use lo+(hi-lo)/2 to avoid overflow in mid.
Time: O(log T) Space: O(1)
MAX_VALUE hi OOB guard unbounded
16 / Feasibility
Find Smallest Divisor Given Threshold
BS on Answer

Divide each element by a positive integer divisor (ceiling), sum the results. Find the smallest divisor where the sum ≤ threshold.

Key Insight
Binary search on divisor in [1 … max(nums)]. Predicate: ⌈nums[i]/div⌉ sum ≤ threshold. Larger divisor → smaller sum → monotonic. Find smallest divisor where ok is true.
Invariant
lo is still an invalid divisor, and hi is a divisor that already satisfies the threshold.
int lo=0, hi=nums[n-1]; // or 1e6
while(lo+1<hi) {
  int mid = lo + (hi-lo)/2;
  if(ok(nums, mid, threshold))
       hi=mid; // seek smaller
  else lo=mid;
}
// ceil: (n+div-1)/div
⚠ Gotcha
Ceiling division trick: (n + div - 1) / div avoids floating point. hi = nums[last] works since max divisor needed ≤ max element. Don't use Math.ceil with int division.
Time: O(n log M) Space: O(1)
minimum valid ceiling division FFTT boundary
17 / Feasibility
Koko Eating Bananas
BS on Speed

N piles of bananas, H hours before guards return. Koko eats k bananas/hour (one pile per hour). Find the minimum k to finish in time.

Key Insight
Binary search on eating speed k ∈ [1 … 10⁹]. Predicate: ∑⌈pile/k⌉ ≤ H. If she can finish at speed k, she can finish at any speed > k (monotonic) → find minimum valid k.
Invariant
lo is too slow to finish in time, and hi is a speed that already works.
int lo=0, hi=(int)1e9;
while(lo+1<hi) {
  int mid = lo + (hi-lo)/2;
  if(ok(piles, mid, H))
       hi=mid; // seek min speed
  else lo=mid;
}
return hi; // first true
⚠ Gotcha
lo must not start at -1 — division by zero when mid=0. Use lo=0. Use ceiling trick (p+mid-1)/mid to avoid float. Return hi (first valid speed, not lo).
Time: O(n log M) Space: O(1)
minimum speed lo≠-1 return hi
18 / Feasibility
Capacity to Ship Within D Days
BS on Capacity

Packages must be shipped in order within D days. Find the minimum ship capacity (max daily load) to accomplish this.

Key Insight
Binary search on ship capacity. Predicate: greedily pack packages sequentially; if adding next package exceeds capacity, start a new day. Check if total days ≤ D. Same structure as Koko.
Invariant
lo is an insufficient capacity, while hi is a capacity that can already ship everything within D days.
// ok(capacity): greedy simulation
int days=1, curr=0;
for(int w : weights) {
  if(w>k) return false; // impossible
  if(curr+w>k){ days++; curr=0; }
  curr+=w;
}
return days<=D;
⚠ Gotcha
Short-circuit: if any single w > capacity, return false immediately. Start days = 1 (not 0). hi can be set to 50000*500 (max packages × max weight) or sum of weights.
Time: O(n log S) Space: O(1)
greedy check days=1 start single item guard
19 / Feasibility
Divide Chocolate (Maximize Min Sweetness)
BS on Answer

Cut a chocolate bar into K+1 pieces. You eat the least-sweet piece. Maximize the sweetness of your piece with optimal cuts.

Key Insight
Inverse of Ship/Koko: maximize minimum piece, not minimize maximum. Predicate: can we cut K+1 pieces each with sweetness ≥ mid? Binary search TTTT'T'FFFF → return lo.
Invariant
lo is a minimum sweetness we can still guarantee, and hi is the first sweetness that is too ambitious.
int lo=1, hi=(int)1e9/persons+1;
while(hi>lo+1) {
  int mid = lo + (hi-lo)/2;
  // TTT'T'FFFF → maximize
  if(ok(s, mid, persons)) lo=mid;
  else                    hi=mid;
}
return lo; // largest valid mid
⚠ Gotcha
Opposite direction from Koko: lo = mid when ok (maximize), return lo. In ok(): accumulate greedily; when chunk ≥ mid, count a cut. If cuts ≥ persons → true.
Time: O(n log S) Space: O(1)
maximize min return lo inverse koko
