2 The Monotonic Predicate ok(mid)
FALSE  ✗
TRUE  ✓
lo
hi

ok() is monotonic — once it flips from false to true (or true to false), it never flips back. This is the fundamental property that makes binary search work on the answer space.

Monotonicity guarantee If ok(x) is true, then ok(x+1) is also true. This means the search space looks like FF…F TT…T — a clean single boundary.
ok(mid) decision logic
Two shapes of the predicate — finding minimum vs maximum valid answer
③ Template
3 Universal Code Template
Python
# ── Template A: find MINIMUM valid (FFFF→TTTT) ──
def solve_min(lo, hi):
  # lo = smallest possible answer
  # hi = largest  possible answer
  while lo < hi:
    mid = lo + (hi - lo) // 2
    if ok(mid):
      hi = mid             # feasible → could be answer
    else:
      lo = mid + 1        # too small → discard
  return lo              # lo == hi == smallest valid

# ── Template B: find MAXIMUM valid (TTTT→FFFF) ──
def solve_max(lo, hi):
  while lo + 1 < hi:
    mid = lo + (hi - lo) // 2
    if ok(mid):
      lo = mid             # feasible → try higher
    else:
      hi = mid
  return lo              # largest valid
Avoid overflowUse lo+(hi-lo)//2 not (lo+hi)//2
hi = midTrue → mid may be answer, don't skip it
lo = mid+1False → mid is wrong, safely skip past it
Return loLoop exits when lo==hi — your answer

④ Convergence
4 How the Range Narrows
Example: Koko Eating Bananas — find minimum eating speed k ∈ [1 … 10⁹].
Each step halves the space → converges in ≈ log₂(10⁹) ≈ 30 iterations.
start
lo=1, hi=10⁹
iter 1
ok=true → hi=mid
iter 2
ok=false → lo=mid+1
iter 3
ok=true → hi=mid
iter 4
ok=false → lo=mid+1
…done
lo == hi → answer ✓
time complexity
O(n · log M)
n = cost of ok() · M = range size. For most problems: O(n log M).
why it works
Monotonicity
The answer space is conceptually sorted (all F then all T), enabling binary search on it.
space complexity
O(1)
Just lo, hi, mid variables. No auxiliary arrays or data structures needed.
